The 6 types of loaders

Not all code and libraries are loaded at program startup, only the ones required for actually running the program. Other libraries are loaded as the program runs, or only as required.

Compile & Go Loader

The assembler itself places the assembled instruction directly into the designated memory locations for execution

General Loader

The loader accepts the assembled machine instructions, data and other information present in the object format and places machine instructions and data in core in an executable computer form.

Absolute Loader

The loader in turn simply accepts the machine language text and places it into core at the location prescribed by the assembler.

Relocating Loaders

To avoid possible reassembling of all subroutines when a single subroutine is changed and to perform the tasks of allocation and linking for the programmer the relocating loaders is introduced.

Direct-linking Loader

A relocatable loader which has advantage of allowing programmer multiple procedure segments and multiple data segments.

Dynamic Loader

a mechanism by which a computer program can, at run time, load a library (or other binary) into memory, retrieve the addresses of functions and variables contained in the library, execute those functions or access those variables, and unload the library from memory.
